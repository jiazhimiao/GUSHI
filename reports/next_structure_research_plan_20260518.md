## 0. 已学到的教训

从 trend_breakout v2 和 cross-sectional alpha 两轮研究得出的约束：

1. **单日单维度信号区分度不足** — HS300 ~230 只股票，日频 alpha 衰减极快 (80%+ next-day out)
2. **A 股个股 alpha 由市场 beta 主导** — 横截面 IC IR 最高仅 0.15，远低于可交易阈值
3. **低波动 ≠ alpha** — 低风险调整后收益无优势
4. **跨年一致性极差** — 仅 1/26 特征在 5 年全部同向
5. **2024 年 924 行情破坏常规 alpha 结构** — 任何依赖"正常市场"的因子在极端行情失效

**因此新方向必须**:
- 不依赖单日单股票 rank 区分度
- 降低市场 beta 影响（对冲/相对逻辑）
- 在极端行情有理论支撑（不仅靠统计）
- 从源头减少候选维度（行业级、事件级、pair 级）

## A. Pair Trading / 行业内相对强弱均值回归

### 核心假设

同一行业内的两只股票，其相对价格比值围绕均值波动。当比值偏离历史均值超过阈值时，**仅做多弱势侧**（A 股无个股做空），等待均值回归。

### ⚠️ A-Share Long-Only 约束

A 股无个股融券做空机制（融券额度极小、成本高、可做空标的有限）。
Pair Trading 不能按 classic long-short market-neutral 实现。

**实际可行方式**:
- 仅做多弱势侧: Z < -2 时买入被低估的 A，不平仓 B
- 不构建完整 pair: 每笔交易是独立的单边做多
- Beta 暴露保留: 没有 short leg 对冲，仍承受市场 beta
- 逻辑仍然是相对价值: 选股基于 pair 内的相对偏离，但执行是 long-only

**这对研究的影响**:
- 回测时不要模拟 short leg（成本、可操作性均不现实）
- 收益归因要区分 pair alpha 和市场 beta
- 与 HS300 基准比较，不是与 0 比较（long-only 天然带 beta）

### 为什么不同于之前的失败方向

| 维度 | trend_breakout v2 | HS300 横截面 | Pair Trading |
|------|-------------------|-------------|--------------|
| 信号源 | 突破绝对值 | 个股 rank | 相对价差 |
| 市场 beta | 完全暴露 | 完全暴露 | 仍有 beta 暴露（long-only 无对冲） |
| 候选维度 | ~300 只 | ~230 只 | ~5-30 对 |
| 区分度问题 | 严重 | 严重 | 每个 pair 独立判断 |
| 极端行情 | 追高/抄底均受损 | 单边失效 | 仅做多弱势侧，熊市亦承压 |

### 需要的数据

- 现有 HS300 日线 OHLCV（已有）
- 行业分类（申万一级 28-31 行业）
  - 通过 AKShare `stock_board_industry_cons_em` 获取
  - **禁止用股票代码前缀推断行业**，必须使用 API 行业分类
- 行业成分历史变化（如可行，否则用最新分类近似，需记录后见偏差风险）

### 最小离线验证方法

```
Phase 1 — Pair 形成（静态）:
  1. 取 HS300 成分股，按行业分组
  2. 行业内按市值/流动性排序，取 top-N 对（相邻市值配对）
  3. 计算每对的历史价差 ratio = log(P_A / P_B)
  4. 计算 ratio 的 rolling mean/std (60d/120d)
  5. Z-score = (ratio - mean) / std

Phase 2 — 信号生成:
  6. Z > +2: B 相对 A 超跌 → 做多 B（只做多，不融券做空）
  7. Z < -2: A 相对 B 超跌 → 做多 A
  8. Z 回归到 ±0.5 以内平仓
  9. 最大持仓 5-10 对，单对仓位上限 5-8%

Phase 3 — 离线统计 (2022-2026):
  10. 记录每笔交易: entry/exit date, pair, direction, return, MAE, duration
  11. 统计: 胜率, avg return, sharpe (近似), max consecutive loss
  12. 年化收益 / 最大回撤
```

### 通过条件

| # | 条件 | 阈值 |
|---|------|------|
| A1 | 胜率 > 55% | t-test vs 50% |
| A2 | 平均每笔收益 > 0 | 扣除 20bps 滑点后 |
| A3 | 最大连续亏损 < 8 笔 | 控制尾部风险 |
| A4 | 4/5 年正收益 | 跨年一致 |
| A5 | 平均持仓 5-20 天 | 不超短也不长持 |
| A6 | 2024 年不崩溃 | 924 行情压力测试 |

### 停止条件

| # | 条件 |
|---|------|
| S1 | 胜率 ≤ 50%（不比随机好） |
| S2 | 行业分类数据缺失或质量不可接受 |
| S3 | 可用 pair < 10（样本不足） |
| S4 | 最大回撤 > 15% |
| S5 | 2022 熊市完全失效 |

### 主要风险

| 风险 | 缓解 |
|------|------|
| A 股无个股做空工具 | 只做多弱势侧，接受 beta 暴露 |
| 行业分类不精确 | 用 AKShare 申万一级分类，**不用代码前缀推断** |
| pair 相关性不稳定 | 只选历史相关性 > 0.7 的 pair |
| 均值回归可能不成立 | Z-score 阈值用 walk-forward 验证 |
| 行业成分随时间变化 | 每季度重建 pair 池 |

### 推荐优先级: ★★★ (最高)

原因: 使用现有数据，最不同于之前的单边排名思路。信号来自行业内 relative value reversion，执行是 long-only（做多弱势侧）。

## B. Industry / Sector Rotation

### 核心假设

A 股行业轮动效应显著。通过行业级别的动量/估值/资金流信号，每月选择 top-N 行业，买入该行业代表性股票或行业 ETF，可以获得超越 HS300 基准的收益。

### 为什么不同于之前的失败方向

| 维度 | HS300 横截面 | Industry Rotation |
|------|-------------|-------------------|
| 候选数 | ~230 只 | ~28-31 个行业 |
| 噪声 | 个股噪声极大 | 行业聚合后信噪比显著提升 |
| 轮动频率 | 日频（过度交易） | 月频（匹配行业轮动节奏） |
| 信号源 | 价格/量因子 | 行业动量 + 宏观/资金流 |

### 需要的数据

- **行业指数日线**: 申万一级行业指数（可通过 AKShare 获取）
- 或自行构建: 用现有 HS300 成分股按行业聚合，构建行业组合日线
- 行业估值数据（可选）: PE/PB 分位数
- 资金流数据（可选）: 行业资金净流入

### 最小离线验证方法

```
Phase 1 — 行业组合构建:
  1. 取 HS300 成分股 + 行业分类
  2. 每日按行业等权聚合，构建 28 个行业日线序列
  3. 计算行业级别特征:
     - ret_20d (行业动量)
     - ret_60d
     - vol_20d
     - industry_advance_decline (行业内涨跌比)
     - relative_to_HS300 (行业超额收益)

Phase 2 — 行业轮动回测:
  4. 每月末 rank 行业，选 top-5
  5. 买入 top-5 行业等权组合
  6. 持有 1 个月后 rebalance
  7. 对比 HS300 基准

Phase 3 — 离线统计 (2022-2026):
  8. 行业轮动组合 vs HS300 基准: 超额收益、信息比率、跟踪误差
  9. 行业集中度、月度换手率
  10. 牛熊分段表现
```

### 通过条件

| # | 条件 | 阈值 |
|---|------|------|
| B1 | 超额收益 > 3% ann vs HS300 | 扣除成本后 |
| B2 | 信息比率 > 0.3 | — |
| B3 | 至少 3/5 年跑赢基准 | — |
| B4 | 最大相对回撤 < 10% | vs HS300 |
| B5 | 2022 熊市跑赢或至少不显著跑输 | 防御性验证 |

### 停止条件

| # | 条件 |
|---|------|
| S1 | 超额收益 ≤ 0（行业轮动无 alpha） |
| S2 | 行业分类数据无法获取 |
| S3 | 信息比率 < 0.1 |
| S4 | 2025-2026 失效（只在早期有效） |

### 主要风险

| 风险 | 缓解 |
|------|------|
| 行业分类边界模糊 | 接受申万标准分类 |
| 行业 ETF vs 自建组合差异 | 优先自建组合，贴近实际可交易 |
| 月频可能错过快速轮动 | 先做月频，后续可扩展到双周 |
| 宏观因子主导、行业 alpha 被淹没 | 控制行业集中度，不押注单行业 |

### 推荐优先级: ★★☆ (中等)

原因: 行业级别信噪比高、逻辑清晰，但需要行业分类数据，且月频验证周期较长。

## C. Event-Driven A 股结构信号 (WAIT)

### 核心假设

A 股存在可预测的事件驱动价格模式。具体事件包括:
1. **指数成分股调整** (HS300/ZZ500 调入/调出)
2. **财报公告** (业绩预告、年报、季报)
3. **股东增减持公告**
4. **回购/分红公告**

这些事件会产生短期资金流冲击或信息传导效应，独立于技术面 alpha。

### 为什么不同于之前的失败方向

| 维度 | HS300 横截面 | Event-Driven |
|------|-------------|--------------|
| 信号源 | 量价因子 | 公司事件 |
| 信号频率 | 每天 | 事件驱动（~1-10/月） |
| 预测逻辑 | 统计相关 | 因果/资金流逻辑 |
| 市场 beta | 无法分离 | 事件日 alpha 可分离 |
| 过拟合风险 | 高（纯数据挖掘） | 低（事件有经济含义） |

### 需要的数据

**已有**:
- 历史成分股调入调出日期 (`data/historical_constituents.json` 中 earliest_inclusion)
- 日线 OHLCV

**需要新增** (均为 AKShare 可获取):
- 业绩预告数据: `ak.stock_yjbb_em(date=...)` — 东方财富业绩预告
- 股东增减持: `ak.stock_ggcg_em(symbol=...)` — 高管增减持
- 回购: `ak.stock_re Purchase_em()` — 回购数据
- 分红: `ak.stock_dividents_cninfo()` — 分红公告
- 或更简单: 用 AKShare 新闻公告接口

### 最小离线验证方法 (先做 C1: 指数调入效应)

```
Phase 1 — 调入效应 (最易验证):
  1. 从 historical_constituents.json 提取 HS300 调入日期
  2. 对每次调入，取调入前后 30 个交易日
  3. 计算:
     - 调入公告日 ~ 生效日之间的 CAR
     - 生效日前后 N 天的超额收益 (vs HS300)
     - 调入后 20d/60d/120d 超额收益
  4. 统计: mean CAR, win rate, 是否具有统计显著性 (t-test)
  5. 比较: 调入 vs 调出股票的表现差异

Phase 2 — 扩展事件 (如 C1 有效):
  6. 业绩预告: 预告日前后 CAR
  7. 高管增持: 公告日后超额收益
  8. 多事件叠加效应

Phase 3 — 简单策略模拟:
  9. 模拟: 调入公告后买入，持有 N 天平仓
  10. 统计年化收益、胜率、最大回撤
```

### 通过条件 (以调入效应为例)

| # | 条件 | 阈值 |
|---|------|------|
| C1 | 调入后 60d 超额收益 > 0 | t-test p < 0.05 |
| C2 | 调入后 win rate > 55% | — |
| C3 | 调入-调出 spread > 0 | 调入跑赢调出 |
| C4 | 2022-2026 各年方向一致 | — |
| C5 | 信号不会随样本增加而消失 | 早期 vs 近期一致 |

### 停止条件

| # | 条件 |
|---|------|
| S1 | 调入效应不显著 (p > 0.1) |
| S2 | 调入后 20d 内无超额收益（已被市场提前消化） |
| S3 | 所需事件数据无法获取 |
| S4 | 效应只在早期存在（2018-2020），近期消失 |

### 主要风险

| 风险 | 缓解 |
|------|------|
| 调入公告日不是实际可交易日 | 区分公告日和生效日 |
| 调入效应被市场提前消化 | 分析公告日 ~ 生效日区间 |
| 幸存者偏差 (只看当前成分股) | 使用 historical_constituents.json 的 earliest_inclusion |
| 事件太少，统计力不足 | 合并多个指数 (HS300+ZZ500) 的调入事件 |
| 未来函数 (用事后成分股倒推) | 只用 earliest_inclusion 字段 |

### 推荐优先级: ★★★ (最高，并列 A)

原因: 已有一部分数据 (index inclusion dates)，逻辑最不同于量价因子，最小验证成本最低（只需分析调入调出日期），如果调入效应不成立可快速放弃。
