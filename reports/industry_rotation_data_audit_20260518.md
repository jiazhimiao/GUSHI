### 1.2 为什么 API 限流

- Eastmoney API (`stock_board_*`, `stock_zh_a_spot_*`) 全部 `ConnectionError: RemoteDisconnected`
- 申万 API (`stock_individual_info_em`) 前 5 次成功，之后全部 `ConnectionError`
- 即使 5 秒间隔 + 批次间 10 秒休息，仍然在第 6 次之后失败
- 这不是单次重试可以解决的问题 — AKShare / Eastmoney 服务器侧限流

## 6. 判定

## C — 行业数据不足，Industry Rotation 暂停

### 理由

1. **行业分类严重不足** — 174/280 (62%) HS300 股票无行业标签
2. **有效行业仅 3 个** — 无法做"轮动"研究 (需要 >= 15 个)
3. **"制造业"占 69%** — 伪行业，内含互不相关的子行业
4. **交易所偏差** — SZ-only 数据不能代表 HS300 整体
5. **SH 蓝筹全部缺失** — 茅台、招行、平安、中信等核心标的无分类
6. **API 全面限流** — 申万/东方财富/中信均不可用

### 不是"暂时不可用"

API 限流已持续 2 天（昨天+今天）。这不是短暂的网络波动。AKShare 对 Eastmoney 和 individual info 端点的访问受到系统性的速率限制。

### 与 Pair A0.5 的相似之处

两个方向都在数据层面被阻塞：
- Pair: 行业分类不足 → 无法做同行业 pair → correlation-based pair 在 2025-2026 失效
- Industry Rotation: 行业分类不足 → 无法构建行业组合 → 无法评估

## 7. 重新打开条件

Industry Rotation 需要以下任一条件满足才能重新打开:

1. **TuShare Pro 账号** — `tushare.pro_api().stock_basic()` 返回申万/中信行业分类，一次性获取全部
2. **AKShare API 恢复** — 非高峰时段重试 `stock_board_industry_cons_em()`
3. **静态行业 CSV** — 从 JoinQuant/RiceQuant/Wind 获取 HS300 申万分类表
4. **Wind/Choice 终端** — 直接提取行业指数日线

最可能路径: **(1) TuShare** — 项目已有 tushare 包，只需 token 即可一次性获取全部行业分类。
