## 1. 为什么 Industry Rotation 可能不同于之前的方向

| 之前的方向 | 失败原因 | Industry Rotation 的区分 |
|-----------|---------|------------------------|
| trend_breakout v2 | 日频个股突破区分度≈0 | 月频行业级别，候选从 ~230→~28 |
| HS300 横截面 rank | IC<0.02，个股噪声主导 | 行业聚合消除个股噪声 |
| Reversal audit | IC IR<0.15，cross-year flips | 行业动量通常更持续 |
| Pair walk-forward | 2025-2026 强趋势中均值回归失效 | 行业轮动可做趋势 + 反转双向 |
| Event-driven | 数据不足 | 行业数据可通过成分股聚合构建 |

**核心假设**: 行业级别的收益序列信噪比 > 个股级别。28-31 个行业候选代替 280 只个股，每个行业是数十只股票的平均，alpha 更稳定。

## 2. 行业分类数据审计

### 2.1 行业分类数据源选择

| 数据源 | 分类体系 | 覆盖 | 可用性 | 历史分类 |
|--------|---------|------|:------:|:------:|
| `ak.stock_info_sz_name_code()` | CSRC 一级 | SZ only | OK | ❌ 仅当前 |
| `ak.stock_individual_info_em()` | 申万二级 | 全部 | ⚠️ 严重限流 | ❌ 仅当前 |
| `ak.stock_board_industry_cons_em()` | 东方财富 | 全部 | ❌ 当前不可用 | ❌ 仅当前 |
| 自建: HS300 成分股聚合 | 申万近似 | ~90%+ | 通过个股 API | ❌ 仅当前 |

### 2.2 行业分类审计清单

B0 第一步必须回答:

1. 可用的行业分类数据源具体是哪个
2. 覆盖 HS300 的比例
3. 分类粒度:
   - CSRC 一级 (19 类) — 过粗，"C 制造业"含 73 只 HS300
   - 申万一级 (28-31 类) — 理想
   - 申万二级 (100+ 类) — 过细，行业内股票太少
4. 是否有历史行业分类数据
5. 如果没有历史分类，使用当前分类回看历史的后见偏差评估:
   - 一家公司的行业归属在 4 年内改变的概率
   - 对行业收益序列的冲击程度

### 2.3 行业分类后见偏差风险

如果只有当前行业分类:
- 2022 年某股票所属的行业可能 ≠ 2026 年
- 公司主业变更、并购重组、ST → 行业跳变
- **缓解**: 申万一级分类变更率极低 (< 5%/年)，大多数蓝筹股行业稳定
- **需要验证**: 取样 10-20 只 HS300 股票，手动确认 2022-2026 申万分类是否变更

## 3. 行业组合构建

### 3.1 构建方法

**方法 A — 成分股聚合（优先）**:
```
1. 每日取 HS300 成分股
2. 按行业分组
3. 行业收益 = 组内股票收益率等权平均
4. 行业指数 = 行业日收益累乘
```
- 优点: 使用现有数据，无需额外 API
- 缺点: 行业分类精度依赖现有分类数据

**方法 B — 申万行业指数 API**:
```
1. 通过 AKShare 获取申万行业指数日线
2. 直接使用标准行业分类
```
- 优点: 标准分类，无歧义
- 缺点: API 当前不可靠

### 3.2 行业组合参数

| 参数 | 值 |
|------|-----|
| 权重方式 | 等权 (避免大市值主导) |
| 最小行业内股票数 | >= 3 |
| 目标行业数 | 15-28 |
| 剔除行业 | 股票数 < 3 或成交额过低 |

### 3.3 审计输出

- 每个行业的日均股票数
- 行业收益序列的完整性 (missing days)
- 行业收益与申万标准行业指数的相关性 (如可获取)
- 行业内股票收益的离散度 (等权 vs 加权差异)

## 5. 离线评估方法

### 5.1 Rebalance 规则

```
频率: 月末
选股: 按特征 rank，选 top-3 / top-5 行业
持仓: 等权持有 top 行业组合
持有期: 1 个月
对比基准: HS300 等权
```

### 5.2 评估指标

| 指标 | 计算 |
|------|------|
| 年化超额收益 | (1+cum_excess)^(12/mon) - 1 |
| 信息比率 (IR) | mean(monthly_excess) / std(monthly_excess) * sqrt(12) |
| 月度 win rate | excess > 0 的月数比例 |
| 最大相对回撤 | max rolling drawdown of cum_excess |
| 跟踪误差 | std(monthly_excess) * sqrt(12) |
| 行业集中度 | top 行业占总仓位的比例 |
| 月度换手率 | 每月更换的行业数 / top-N |

### 5.3 分段评估

| 分段 | 目的 |
|------|------|
| 2022 熊市 | 行业轮动能否防御 |
| 2023 震荡 | 轮动是否有效 |
| 2024 924 前后 | 极端行情稳定性 |
| 2025-2026 | 近期是否延续 (Pair 在此段失败) |

## 8. B0 审计步骤 (最小实施)

```
Step 1 — 行业分类 (0.5 天):
  1. 取 HS300 成分股当前行业分类
  2. 统计行业分布和覆盖率
  3. 检查历史行业分类变更可能性

  4. 每日按行业等权聚合 HS300 成分股
  5. 构建 2022-2026 行业日收益序列
  6. 检查数据质量 (缺失、跳变、股票数不足)

Step 3 — 特征计算 (0.5 天):
  7. 计算 8 个行业特征
  8. 月末 rank → top-3/5 行业选择
  
Step 4 — 离线评估 (1 天):
  9. 月度 rebalance 模拟
  10. 超额收益 + IR + 分段评估
  11. 输出 pass/stop verdict
```
